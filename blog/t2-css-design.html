Relative, absolute, and fixed positioning...What's the difference?

A brief summary on how to use these important CSS attributes

3-29-2015

If you're new to HTML and CSS, chances are you've driven youself crazy trying to position HTML elements in a manner that makes your page look stylish, clean, organized and easy to use. If you haven't, then I applaud you: you're a natural web designer! The rest of us, however, could use a brief tip on how to position HTML elements neatly on a web page using the static, absolute, relative, and fixed positioning types in CSS. So whadaya say? Let's get started?

First, let's go over what it means for an element's position property to be static. Static positioning basically means that your HTML elements would follow a "natural order", so to speak. For example, if you were to create an orange square and a blue square without setting any position properties, it would default to position: static, and you would see an blue square with an orange square stacked on top of it, like this:

Notice that I specified each element's position property to static? Hence the natural stacked order of things. Unlike Warren Buffett, we're more ambitious than that. So, let's go ahead and set each div's position property to absolute.

What happened? Where did the blue square go? The answer is simple. Absolute positioning positions an element in relation to the first parent element it has that does not have the position property of static. If there is no parent element, then it gets positioned relative to the html element (relative to the entire html document). Since both of the squares are children of html, they will both be positioned one behind the other in the upper left corner.

Now, let's try something different. Let's set the position properties of our squares to relative and set the orange square's position from the lef to left:2em and see what happens.

See how the orange square shifted to the right by 2em? That's because relative positioning positions an element relative to its normal position, i.e. where it would be if you didn't assign it any position property. So, the combination of assigning to an element the position of relative and then specifying a 2em distance from the left would shift the element 2em from where it would normally be. Pretty straight-forward huh?

Now let's move on to try something really cool. Let's say that you want to have a sidebar with links to your other blog posts positioned on the left of the page, but you don't want it to move out of the screen when a user scrolls down. Instead, you want it to stay on the screen regardless of whether or not the user scrolls down the page. In this case, you would use fixed positioning. When an item is assigned the position property of fixed, it is positioned relative to the browser window and will not move even if you scroll.

I hope this was helpful and that you now understand the difference between the different positioning properties in CSS. Ciao!

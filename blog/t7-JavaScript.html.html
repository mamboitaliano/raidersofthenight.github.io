Ruby vs. JavaScript

How do they differ?

May 1st, 2015

By now we've spent four weeks learning Ruby. Now it's on to JavaScript. During this last week, we hit it hard a learned as much as one could learn about JavaScript in seven days. Now it's time to sit back and reflect on how both languages differ. First and foremost, it is imporant to not that Ruby is a class-based language whereas Javascript is considered a prototype-based language. Ruby allows us to organize classes to call instance methods on objects of that class and manipulate data. However, Object Oriented Programming can also be...well...simulated in JavaScript using functions to mimic classes.

One way to do this is by using the constructor function on a new object in Javascript, which allows that object to inherit the prototype methods associated with that function. While Ruby uses class methods, JavaScript uses static methods which can't be called on an object but can be called on the function itself.

When it comes to declaring a class in Ruby, we use the syntax class Terminator followed by the end of input keyword, end. In JavaScript, however, declaring a class is done with the syntax function Terminator(modelNo, modelType) { };

Instance methods in Ruby are defined using the syntax def method_name ...end, while in JavaScript, the sytax is Classname.prototype.methodName = function() {};

Class methods in Ruby are defined using the syntax def self.method_name ...end, while in JavaScript, the sytax is Classname.methodName = function() {};

Looping is another thing that is vastly different between Ruby and JavaScript. A typical Ruby loop will look like this: my_array.each { |i| do something...}, where i is a variable that holds the value of the current iteration. For example, if you have array my_array[1, 2, 3, 4, 5], the .each part tells Ruby that this is a loop, and that it's going to have to iterate through my_array, holding the value of each index in i during each iteration. The part after the |i| is where you put the code that tells Ruby what to do during each iteration.

On the other hand, looping in JavaScript follows nearly the same syntax and logic as most C languages and Java. The most common JavaScript loop is the for loop. This is what a typical for loop in JavaScript looks like: for (i = 0; i < my_array.length; i++) { }, where i = 0 tells JavaScript to begin at 0, i < my_array.length tells JavaScript to iterate through my_array as long as i is less than the length of my_array, and finally to increment i by 1 during every iteration (i++).
